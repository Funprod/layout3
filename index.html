Введение

JavaScript - кроссплатформенный объектно-ориентированный скриптовый язык. Это небольшой и легковесный язык. Внутри хост-среды (например, веб-браузера) JavaScript может быть подключен к объектам своей среды для обеспечения программного контроля над ними.

JavaScript содержит стандартную библиотеку объектов, таких как Array, Date и Math, а также базовый набор языковых элементов, таких как операторы, управляющие структуры и операторы. Базовый JavaScript может быть расширен для различных целей путем дополнения его дополнительными объектами; например:

  • Клиентский JavaScript расширяет базовый язык, предоставляя объекты для управления браузером и его объектной моделью документа (DOM). Например, расширения на стороне клиента позволяют приложению размещать элементы в HTML-форме и реагировать на события пользователя, такие как щелчки мыши, ввод формы и навигация по странице.
  • Серверный JavaScript расширяет базовый язык, предоставляя объекты, имеющие отношение к запуску JavaScript на сервере. Например, серверные расширения позволяют приложению взаимодействовать с базой данных, обеспечивают непрерывность передачи информации от одного вызова приложения к другому или выполняют манипуляции с файлами на сервере.

JavaScript и Java

JavaScript и Java похожи в некоторых отношениях, но принципиально отличаются в некоторых других. Язык JavaScript напоминает Java, но не имеет статической типизации Java и строгой проверки типов. JavaScript следует большинству синтаксисов выражений Java, соглашениям об именовании и базовым конструкциям потока управления, что и стало причиной переименования его из LiveScript в JavaScript.

В отличие от системы классов Java во время компиляции, построенной с помощью объявлений, JavaScript поддерживает систему выполнения, основанную на небольшом количестве типов данных, представляющих числовые, логические и строковые значения. JavaScript использует объектную модель на основе прототипа вместо более распространенной объектной модели на основе классов. Модель на основе прототипа обеспечивает динамическое наследование; то есть то, что наследуется, может варьироваться для отдельных объектов. JavaScript также поддерживает функции без каких-либо специальных декларативных требований. Функции могут быть свойствами объектов, выполняемых как слабо типизированные методы.

JavaScript - язык с очень свободной формой по сравнению с Java. Вам не нужно объявлять все переменные, классы и методы. Вам не нужно беспокоиться о том, являются ли методы общедоступными, частными или защищенными, и вам не нужно реализовывать интерфейсы. Переменные, параметры и типы возвращаемых функций явно не вводятся.

Переменные

Переменные используются в качестве символических имен для значений в вашем приложении. Имена переменных, называемые идентификаторами, соответствуют определенным правилам.

Идентификатор JavaScript должен начинаться с буквы, подчеркивания (_) или знака доллара ($); последующие символы также могут быть цифрами (0-9). Поскольку JavaScript чувствителен к регистру, буквы включают символы от "A" до "Z" (в верхнем регистре) и символы от "a" до "z" (в нижнем регистре).

В идентификаторах можно использовать буквы стандарта ISO 8859-1 или Unicode, такие как å и ü. Вы также можете использовать escape-последовательности Unicode в качестве символов в идентификаторах. Некоторыми примерами юридических названий являются Number_hits, temp99 и _name.

Объявление переменных
Вы можете объявить переменную тремя способами:

С ключевым словом var. Например,var x = 42.этот синтаксис может использоваться для объявления как локальных, так и глобальных переменных.

Просто присвоив ей значение. Например,x = 42.это всегда объявляет глобальную переменную. Это генерирует строгое предупреждение JavaScript. Вы не должны использовать этот вариант.

С ключевым словом let . Например,let y = 13.этот синтаксис можно использовать для объявления локальной переменной области видимости блока. Смотрите Область видимости переменной ниже.

Область действия переменной

Когда вы объявляете переменную вне какой-либо функции, она называется глобальной переменной, потому что она доступна для любого другого кода в текущем документе. Когда вы объявляете переменную внутри функции, она называется локальной переменной, потому что она доступна только внутри этой функции.

JavaScript до ECMAScript 2015 не имел области действия оператора block; скорее, переменная, объявленная внутри блока, является локальной по отношению к функции (или глобальной области действия), в которой находится блок. Например, следующий код будет регистрировать 5, потому что область действия x - это функция (или глобальный контекст), в пределах которой объявлен x, а не блок, который в данном случае является оператором if.

if (true) { var x = 5; } console.log(x); // 5

Это поведение меняется при использовании объявления let, представленного в ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
